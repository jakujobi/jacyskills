# Repo Documentation Design

**Goal:** Add minimal public-facing documentation to make the `jacyskills` repo understandable and installable from GitHub without adding extra repo clutter.

## Scope

Add one root `README.md` only.

Do not add per-skill `README.md` files.

## Content

The root `README.md` should include:

1. repo purpose
2. included skills
3. global install commands
4. repo layout
5. note that AMBER-specific skills are intentionally excluded

## Rationale

This repo is intentionally small and focused. A single root README gives a clean GitHub landing page, documents installation, and avoids duplicating information already present in each `SKILL.md`.
